== 2. Neo4j Desktop : Configuration, Exécution et Visualisation

NOTE: Ce guide s'appuie sur la **documentation APOC Extended (v2026)** et la **documentation Neo4j Desktop (v2.1)**.

=== Étape 2.1 : Configuration de l'Environnement
Avant d'exécuter des scripts, vérifiez que votre environnement Neo4j Desktop est entièrement configuré à l'aide de la documentation officielle :

* **https://neo4j.com/docs/desktop/current/operations/instance-management[Créer une Instance Locale] :** Suivez les étapes pour créer un nouveau SGBD local.

NOTE: le répertoire https://neo4j.com/docs/desktop/current/operations/instance-management/#_data_path[$NEO4J_HOME] est directement accessible depuis la carte de votre instance via `Open Folder`.

* **https://neo4j.com/labs/apoc/2026/installation/#neo4j-desktop[Installer le Plugin APOC extended] :** APOC est requis pour l'exécution avancée de scripts. Prenez note de la https://neo4j.com/labs/apoc/2026/installation/#_version_compatibility_matrix[Matrice de compatibilité des versions].

NOTE: si le fichier `apoc.conf` n'existe pas dans le répertoire de configuration (`$NEO4J_HOME/conf/`), créez-le et modifiez-le comme indiqué.

* **Activer la Lecture de Fichiers :** Vous devez https://neo4j.com/labs/apoc/2026/overview/apoc.cypher/apoc.cypher.runFile/#_reading_from_a_file[autoriser APOC à lire des fichiers locaux].

=== Étape 2.2 : Création de la Base de Données et Exécution du Script
Une fois l'environnement correctement configuré :

1. **Créer la Base de Données Cible :** https://neo4j.com/docs/desktop/current/operations/instance-management/#_startstoprestart_instance[Démarrez votre instance] et https://neo4j.com/docs/desktop/current/operations/database-management/#_create_a_new_database[créez la base de données] qui hébergera votre graphe (ex. ontograph | relgraph | ontorelgraph).
2. **Préparer le Script :** Déplacez votre script cible (par ex. `OntoRelGraph.cypher`) dans le répertoire d'importation de la base de données (`$NEO4J_HOME/import/`).
3. **Exécuter le Script :** Basculez vers https://neo4j.com/docs/desktop/current/query/[l'interface **Query**]. https://neo4j.com/docs/desktop/current/query/#:~:text=Connect%20to%20your%20instance%20via%20the%20connection%20dropdown%20and%20select%20your%20database.[Connectez-vous à votre instance et sélectionnez votre base de données]. Exécutez la commande suivante pour lancer votre fichier :
+
[source,cypher]
----
CALL apoc.cypher.runFile("OntoRelGraph.cypher")
----
_(Référence : https://neo4j.com/labs/apoc/2026/overview/apoc.cypher/apoc.cypher.runFile/#usage-apoc.cypher.runFile[Utilisation de apoc.cypher.runFile])_

=== Étape 2.3 : Visualisation des Sous-Graphes Démos
Une fois que le script a été exécuté avec succès et a généré votre graphe, vous pouvez filtrer et visualiser un sous-graphe spécifique.

Selon si vous avez roulé OntoRelGraph.cypher pour MONDIAL ou MPHPO copiez la requête corrspondante ci-dessous, collez-la directement dans https://neo4j.com/docs/desktop/current/query/#_cypher_editor[l'editeur Cypher], puis appuyez sur `Ctrl + Entrée` (ou cliquez sur le bouton Lecture) :

==== MONDIAL
[source,cypher]
----
WITH [
  'decimal',
  'Measurement_value_decimal',
  'Measurement',
  'YearlyMeasurement',
  'YearlyMeasurement_year_Literal',
  'Literal',
  'YearlyMeasurement_year_nonNegativeInteger',
  'nonNegativeInteger',
  'PopulationCount_population_nonNegativeInteger',
  'PopulationCount',
  'hadPopulation',
  'Country_hadPopulation_PopulationCount',
  'Country'
] AS targetIds
MATCH (n)-[r]->(m)
WHERE n.relId IN targetIds AND m.relId IN targetIds
RETURN n, r, m
----

==== MPHPO
[source,cypher]
----
WITH [
  'string',
  'MPHPO_0000006_Ontorel-Core_0000002_string',
  'MPHPO_0000006',
  'HOSO_0000084',
  'HEPRO_0000010_IAO_0000219_NCBITaxon_9606',
  'IAO_0000219',
  'HOSO_0000084_IAO_0000219_HOSO_0000079',
  'HEPRO_0000009',
  'HEPRO_0000010',
  'NCBITaxon_9606'
] AS targetIds
MATCH (n)-[r]->(m)
WHERE n.relId IN targetIds AND m.relId IN targetIds
RETURN n, r, m
----

== 3. AgensGraph : Création de l'Instance et Exécution du Script

=== Étape 3.1 : Initialisation de la Base de Données et du Schéma de Graphe
1. Ouvrez le terminal de votre système ou votre invite de commande.
2. Accédez à l'outil d'interface en ligne de commande interactif (`agens`) en exécutant la commande de connexion (ajustez les paramètres pour votre utilisateur personnalisé si nécessaire) :
+
[source,bash]
----
agens -U postgres -d postgres
----
3. Initialisez une base de données propre destinée au stockage de votre graphe :
+
[source,sql]
----
CREATE DATABASE agens;
\c agens
----
4. Créez votre objet de schéma de graphe de propriétés cible, puis ajustez le chemin d'accès d'exécution pour en faire votre espace de travail actif :
+
[source,sql]
----
CREATE GRAPH ontorelgraph;
SET graph_path = ontorelgraph;
----

=== Étape 3.2 : Exécution du Script openCypher
1. Quittez le shell interactif ou ouvrez un nouvel onglet de terminal. Exécutez votre fichier de script openCypher directement sur votre schéma fraîchement configuré à l'aide de l'option `-f` :
+
[source,bash]
----
agens -U postgres -d agens -f /chemin/vers/votre/OntoRelGraph.cypher
----
+
_(Alternativement, vous pouvez copier les clauses de votre script et les coller directement dans votre invite interactive `agens` active après avoir défini le `graph_path`)._
