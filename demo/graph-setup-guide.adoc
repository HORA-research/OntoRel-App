== 2. Neo4j Desktop: Setup, Execution, and Visualization

NOTE: This guide is based on the **APOC Extended Documentation (v2026)** and the **Neo4j Desktop Documentation (v2.1)**.

=== Step 2.1: Environment Configuration
Before executing scripts, verify your Neo4j Desktop environment is fully configured using the official documentation:

* **https://neo4j.com/docs/desktop/current/operations/instance-management/[Create a Local Instance] :** Follow the steps to create a new local DBMS.

NOTE: https://neo4j.com/docs/desktop/current/operations/instance-management/#_data_path[$NEO4J_HOME] can be directly accessed from your instance card through `Open Folder`.

* **https://neo4j.com/labs/apoc/2026/installation/#neo4j-desktop[Install the APOC extended Plugin] :** APOC is required for advanced script execution. Take note of the https://neo4j.com/labs/apoc/2026/installation/#_version_compatibility_matrix[Version Compatibility Matrix].
* **Enable File Reading :** You must https://neo4j.com/labs/apoc/2026/overview/apoc.cypher/apoc.cypher.runFile/#_reading_from_a_file[allow APOC to read local files].

NOTE: if `apoc.conf` does not exist in config directory (`$NEO4J_HOME/conf/`), create it and edit as indicated.

=== Step 2.2: Database Creation & Script Execution
Once the environment is properly configured:

1. **Create the Target Database:** https://neo4j.com/docs/desktop/current/operations/instance-management/#_startstoprestart_instance[Start your instance] and https://neo4j.com/docs/desktop/current/operations/database-management/#_create_a_new_database[create the database] that will hold your graph (e.g., ontograph | relgraph | ontorelgraph).
2. **Prepare the Script:** Move your target script (e.g., `OntoRelGraph.cypher`) into the database's import directory (`$NEO4J_HOME/import/`).
3. **Execute the Script:** Switch to the https://neo4j.com/docs/desktop/current/query/[**Query** interface]. https://neo4j.com/docs/desktop/current/query/#:~:text=Connect%20to%20your%20instance%20via%20the%20connection%20dropdown%20and%20select%20your%20database.[Connect to your instance and select your database]. Run the following command to execute your file:
+
[source,cypher]
----
CALL apoc.cypher.runFile("OntoRelGraph.cypher")
----
_(Reference: https://neo4j.com/labs/apoc/2026/overview/apoc.cypher/apoc.cypher.runFile/#usage-apoc.cypher.runFile[apoc.cypher.runFile Usage])_

=== Step 2.3: Visualizing the Demo Subgraphs
After the script has successfully executed and built your graph, you can filter and visualize a specific subgraph.

Depending on whether you have run OntoRelGraph.cypher for MONDIAL or MPHPO, copy the corresponding query below, paste it directly into the https://neo4j.com/docs/desktop/current/query/#_cypher_editor[Cypher editor] of the **Query** interface, and press `Ctrl + Enter` (or click the Play button):

==== MONDIAL
[source,cypher]
----
WITH [
  'decimal',
  'Measurement_value_decimal',
  'Measurement',
  'YearlyMeasurement',
  'YearlyMeasurement_year_Literal',
  'Literal',
  'YearlyMeasurement_year_nonNegativeInteger',
  'nonNegativeInteger',
  'PopulationCount_population_nonNegativeInteger',
  'PopulationCount',
  'hadPopulation',
  'Country_hadPopulation_PopulationCount',
  'Country'
] AS targetIds
MATCH (n)-[r]->(m)
WHERE n.relId IN targetIds AND m.relId IN targetIds
RETURN n, r, m
----

==== MPHPO
[source,cypher]
----
WITH [
  'string',
  'MPHPO_0000006_Ontorel-Core_0000002_string',
  'MPHPO_0000006',
  'HOSO_0000084',
  'HEPRO_0000010_IAO_0000219_NCBITaxon_9606',
  'IAO_0000219',
  'HOSO_0000084_IAO_0000219_HOSO_0000079',
  'HEPRO_0000009',
  'HEPRO_0000010',
  'NCBITaxon_9606'
] AS targetIds
MATCH (n)-[r]->(m)
WHERE n.relId IN targetIds AND m.relId IN targetIds

----

== 3. AgensGraph: Instance Creation & Script Execution

=== Step 3.1: Initialize Database and Graph Schema
1. Open your system terminal or command-line interface.
2. Access the interactive CLI tool (`agens`) by running the connection command (adjusting parameters for your custom user if necessary):
+
[source,bash]
----
agens -U postgres -d postgres
----
3. Initialize a clean database intended for your graph storage:
+
[source,sql]
----
CREATE DATABASE agens;
\c agens
----
4. Create your target property graph schema object, and adjust the runtime path to make it your active operating workspace:
+
[source,sql]
----
CREATE GRAPH ontorelgraph;
SET graph_path = ontorelgraph;
----

=== Step 3.2: Execute openCypher Script
1. Exit the interactive shell or open a new terminal tab. Run your structured openCypher script file directly against your newly configured schema using the `-f` flag:
+
[source,bash]
----
agens -U postgres -d agens -f /path/to/your/OntoRelGraph.cypher
----
+
_(Alternatively, you can copy your script clauses and paste them directly into your active interactive `agens` prompt line after setting the `graph_path`)._
